About

The Large-Scale Test Data Browser (also known as Prototype Large-Scale Accessibility Conformance Monitoring, or PLACM) is a deliverable (D3.3) of the WAI-Tools project, a European Commission (EC) co-funded project under the Horizon 2020 Program (780057).

The objective of this prototype is to demonstrate how existing standards and open-data formats allow for the development of tools to assist in the study and monitoring of large amounts of accessibility evaluation data.

What data is supported

PLACM allows visualizing data from two categories of evaluation results: assertions and success criteria. The data that can be visualized for each category is:

  • Assertions
    • Number of pages;
    • Number of assertions with result 'passed';
    • Number of assertions with result 'failed';
    • Number of assertions with result 'can't tell';
    • Number of assertions with result 'inapplicable';
    • Number of assertions with result 'untested';
  • Success criteria
    • Number of criteria with result 'passed';
    • Number of criteria with result 'failed';
    • Number of criteria with result 'can't tell';
    • Number of criteria with result 'inapplicable';
    • Number of criteria with result 'untested'.

This data can be grouped by different categories of metadata, from country to ACT rule assessed. The different categories supported can be found in the sidebar.

What can I do

PLACM allows navigating and browsing the data in different ways. The simplest one is just selecting one category of data from the sidebar to access all the data grouped by the category.

The other visualization options can be accessed by selecting one of the bars in a chart. After selecting a bar, a window will be displayed where one of the following visualization modes can be selected

  • Drilldown allows consulting data from a category below, filtered by the data points corresponding to the bar selected. Multiple drilldowns can be applied consecutively.
  • Group by allows comparing the data points of the bar selected, grouped by entries of a category above.
  • Comparison allows comparing the data points of the bar selected with other entries of the same category.
  • Timeline displays the temporal evolution of data by month. Each point in the horizontal axis is a month and the corresponding data are the results evaluated in that month.
  • Details allows to consult the assertions contributing to the outcome of each success criterion. This visualization is only accessible if the visualization in which the modal window was opened is based on the category 'Application/Website' and the results shown are of success criteria. This navigation generates an accordion, with success criteria grouped by their result

Below each bar chart there is a checkbox list that allows toggling the visibility of specific data points.

Above this list there is a button that toggles the data presented between assertions and success criteria. Success criteria visualizations are only possible for charts displaying data for the Success criteria category and above.

Entering data

Data can be entered into the platform through the administration page. Two types of data sources have been considered: Accessibility Statements and EARL Reports. Documents are parsed to store application-related data and test results, in the form of assertions. After selecting a file (or multiple files), a form is presented where metadata about the report(s) being uploaded can be provided such that the data can be browsed according to the existing categories.
