About

This Prototype Large-Scale Accessibility Conformance Monitoring (PLACM) demonstrates the use of open data to support large-scale accessibility monitoring initiatives. This includes monitoring conformance with requirements of the EU Web Accessibility Directive (WAD).

What does this tool do?

PLACM visualizes accessibility data that could be scraped from:

  • Accessibility reports, such as those created with the W3C WCAG-EM Report Tool
  • Accessibility statements, such as those created with the W3C Accessibility Statements Generator

PLACM allows you to browse such accessibility data in a variety of ways, for example to group and compare different countries, industry sectors, and other parameters. It simulates large-scale monitoring of accessibility data that is drawn from decentralized sources, for example from accessibility reports and statements that website owners make available to the monitoring body.

How does this tool work?

PLACM allows you to browse accessibility data in different ways. One option is to select just one category of data from the sidebar to access all the data grouped by that category.

Other visualization options can be accessed by selecting one of the bars in a chart. After selecting a bar, a window is displayed where one of the following visualization modes can be selected:

  • Drilldown allows you to consult data from a category below, filtered by the data points corresponding to the bar selected. Multiple drilldowns can be applied consecutively.
  • Group by allows you to compare the data points of the bar selected, grouped by entries of a category above.
  • Comparison allows you to compare the data points of the bar selected with other entries of the same category.
  • Timeline displays the temporal evolution of data by month. Each point in the horizontal axis is a month and the corresponding data are the results evaluated in that month.
  • Details allows you to consult the assertions contributing to the outcome of each success criterion. This visualization is only available when the visualization in which the modal window was opened is based on the category 'Application/Website' and the results shown are of success criteria. This navigation generates an accordion, with success criteria grouped by their result.

Below each bar chart there is a checkbox list that allows toggling the visibility of specific data points.

Above this list there is a button that toggles the data presented between assertions and success criteria. Success criteria visualizations are only possible for charts displaying data for the Success criteria category and above.

Which data is supported?

PLACM allows you to visualize data from two categories of evaluation results: assertions and success criteria. The data that can be visualized for each category is:

  • Assertions
    • Number of pages;
    • Number of assertions with result 'passed';
    • Number of assertions with result 'failed';
    • Number of assertions with result 'can't tell';
    • Number of assertions with result 'inapplicable';
    • Number of assertions with result 'untested';
  • Success criteria
    • Number of criteria with result 'passed';
    • Number of criteria with result 'failed';
    • Number of criteria with result 'can't tell';
    • Number of criteria with result 'inapplicable';
    • Number of criteria with result 'untested'.

How is the data entered?

PLACM uses the machine-readable JSON format. It relies on accessibility reports with data that is structured according to the W3C Evaluation and Report Language (EARL). See also the GitHub EARL repository. PLACM scrapes additional information from accessibility statements.

The currently available data has fictional yet realistic values. You can enter additional data into the platform through the administration page. Two types of data sources are considered: Accessibility Statements and EARL Reports. The tool parses the documents, and stores application-related data and test results, in the form of assertions. After selecting a file or multiple files, a form is presented where metadata about the report(s) being uploaded can be provided, so that the data can be browsed according to the existing categories.
